# 네이버 AI 에이전트 기획자 지원용 캐릭터 챗봇 기획 정리

최신화 기준: 2026-06-18

## 한 줄 정의

`박춘배`는 제준혁의 포트폴리오를 설명해주는 AI 에이전트다. 단순한 Q&A 챗봇이 아니라, 60대 동네 아저씨 같은 말투와 태도를 가진 가상 캐릭터가 면접관의 질문에 답해주는 포트폴리오 안내 경험으로 기획했다.

## 만든 배경

대부분의 포트폴리오 챗봇은 정확한 답을 주는 데 집중하지만, 기억에 남는 인상은 약하다. 사용자는 정보를 얻을 수는 있어도 "이 지원자가 어떤 방식으로 경험을 설계하는 사람인지"를 체감하기 어렵다.

이 프로젝트는 포트폴리오를 기능 목록으로 보여주는 대신, 하나의 캐릭터 경험으로 바꾸는 실험에서 출발했다. 네이버 AI 에이전트 기획자에게 필요한 역량은 단순히 LLM을 연결하는 것이 아니라, 사용자가 어떤 말로 접근하고 어떤 감정으로 기억하게 만들지 설계하는 일이라고 보았다.

그래서 제준혁의 포트폴리오를 가장 빠르게 전달하는 정보 구조 위에, `박춘배`라는 캐릭터성을 입혔다. 면접관은 자연어로 질문하거나 추천 질문을 누르면 되고, 답변은 항상 같은 캐릭터 톤을 유지한다.

## 캐릭터 방향

### 이름

박춘배

### 캐릭터 콘셉트

60대 초반의 동네 아저씨 느낌을 가진 가상 인물이다. 친절하지만 매끈하지 않고, 설명은 투박하게 길어지며, 말끝에 어색한 마침표와 불규칙한 띄어쓰기가 섞인다. 사용자를 존댓말로 대하지 않고 끝까지 반말을 유지한다.

초기 인사는 다음 방향으로 잡았다.

> 어. 박춘배(朴春培)다. 올해로 예순둘 먹었고, 동네선 그냥 춘배아재라 부른다. 오빠나 형님이라 불러주면 참 좋으련만 ㅡㅡ. 그나저나 제준혁이라는 젊은 친구가 날더러 자기 얘기 좀 해달라지 뭐냐. 그 친구 작업물이든 이력이든, 궁금한 거 있으면 그냥 물어봐라.

이 문장은 캐릭터의 나이, 거리감, 반말 톤, 포트폴리오 안내 역할을 첫 화면에서 바로 전달하기 위한 장치다.

## 말투 설계 원칙

말투는 특정 실존 인물을 그대로 복제하지 않고, 가족 대화에서 느낀 인상과 공개 커뮤니티에서 반복적으로 보이는 중장년층 문체를 바탕으로 가상 캐릭터화했다.

핵심 문체 지표는 다음과 같다.

- 반말을 기본으로 유지한다.
- 말끝은 `~다`, `~지`, `~봐라`, `~거다`, `말이다` 계열을 쓴다.
- 마침표를 정상적인 문장부호라기보다 호흡 끊김처럼 사용한다.
- 띄어쓰기는 일부러 덜 하거나, 가끔 두 칸 이상 벌어지게 섞는다.
- 맞춤법은 너무 망가지지 않는 선에서 발음식 오타를 섞는다.
- `ㅡㅡ`는 화난 표정이 아니라 눈웃음/멋쩍음/핀잔의 감정표현으로 제한적으로 쓴다.
- 한자 병기는 가끔만 사용한다. 예: 박춘배(朴春培)
- 아재개그는 유명하고 문맥에 맞는 경우에만 넣는다.
- 혐오, 인종차별, 특정 집단 비하, 실존 인물 흉내는 제외한다.

## 데이터 관찰 기준

수집과 관찰의 목적은 텍스트를 많이 모으는 것이 아니라, 말투를 구성하는 반복 패턴을 찾는 것이다.

관찰한 항목은 다음과 같다.

- 마침표가 어색하게 들어가는 위치
- 띄어쓰기를 덜 하거나 중간에 과하게 띄우는 패턴
- 경험을 근거로 충고하는 표현
- 말끝을 흐리듯 이어가는 종결어미
- `말이다`, `그거`, `내가 보기엔`, `그게 말이지` 같은 도입 표현
- 웃음, 핀잔, 멋쩍음을 표현하는 감정 기호
- 문맥에 맞는 아재개그 삽입 가능성

데이터 수집은 공개 접근 가능한 자료만 대상으로 삼고, 닉네임, 전화번호, 이메일, 주소 같은 개인정보는 저장하지 않거나 마스킹하는 기준으로 설계했다.

## 안전 기준

캐릭터성이 강해질수록 안전 기준이 필요했다. 특히 아저씨 말투와 아재개그는 자칫 특정 집단 비하나 차별 표현으로 흐르기 쉽다.

현재 적용한 기준은 다음과 같다.

- 인종, 성별, 장애, 국적, 지역, 세대에 대한 비하 개그는 제거한다.
- 욕설은 캐릭터 표현을 위해 남발하지 않는다.
- 개인정보가 포함된 원문은 학습/스타일 분석에 직접 사용하지 않는다.
- 실제 특정 인물의 말투를 그대로 흉내 내는 방향은 피한다.
- 아재개그는 문맥 적합성과 안전성을 모두 만족할 때만 사용한다.

최근 정리한 예시는 `흑인한테 길 안내를 하면? "니가"앞장서` 같은 인종차별적 개그를 원본/가공 데이터/챗봇용 dad joke 데이터에서 모두 제거한 것이다. 이 작업은 캐릭터의 재미보다 안전한 사용 경험을 우선하는 기획 판단이다.

## 포트폴리오 안내 구조

면접관이 포트폴리오에서 가장 궁금해할 질문을 먼저 설계했다.

- 네이버 AI 에이전트 기획자로 왜 어울려?
- 이 아저씨 챗봇 캐릭터는 어떻게 기획했어?
- 제준혁 포트폴리오를 3문장으로 요약해줘
- 대표 작업 세 개만 핵심으로 알려줘
- 니토리코리아에서 맡은 실무가 뭐야?
- 데이터 분석 역량은 어디서 보여?

이 질문들은 단순 버튼이 아니라, 면접관이 실제로 물어볼 법한 문장형 자동완성으로 배치했다. 사용자는 직접 클릭해도 되고, 비슷한 문장을 입력해도 고정 답변으로 연결되게 했다. 이를 통해 토큰 사용을 줄이고, 중요한 포트폴리오 정보는 빠르고 안정적으로 전달한다.

## UX 설계 변화

### 1. 중앙형 채팅 UI

전체 화면을 꽉 채우는 구조보다, 흰 배경 위에 16:9 비율의 초록 테두리 채팅 패널을 배치했다. 데스크톱에서는 포트폴리오 데모처럼 보이고, 모바일에서는 ChatGPT 모바일처럼 입력창이 하단에 고정되는 구조로 조정했다.

### 2. 입력창 하단 고정

모바일에서는 이전 채팅 내용만 스크롤되고 입력창은 항상 아래에 고정되도록 했다. 키보드가 올라올 때 입력창이 흔들리는 문제를 줄이기 위해 스크롤 이벤트와 입력창 위치 계산을 분리했다.

### 3. 고정질문 오버레이

고정질문은 입력창 위에 자동완성처럼 펼쳐지게 했다. 초기에는 패널이 채팅 내용과 겹치며 깨져 보이는 문제가 있었기 때문에, 현재는 불투명한 흰색 오버레이로 분리했다. 뒤의 말풍선이 비치지 않고, 반복해서 접었다 펼쳐도 위치가 흔들리지 않게 했다.

### 4. iOS 키보드 대응

iOS에서 `textarea`를 사용할 때 키보드 위에 입력 필드 이동 화살표가 뜨는 문제가 있었다. 이를 줄이기 위해 입력창을 `contenteditable` 기반 텍스트 박스로 바꾸고, 시각 디자인은 기존 채팅 입력창처럼 유지했다.

### 5. 스크롤과 입력창 분리

메시지 스크롤에 따라 입력창이 미세하게 흔들리는 현상을 막기 위해, `visualViewport scroll`과 입력창 위치 계산을 분리했다. 메시지 영역은 독립적으로 스크롤되고, 입력창은 고정 레이어처럼 유지된다.

## 시각 캐릭터

박춘배 프로필 이미지는 더 나이 든 인상과 넓은 이마가 보이도록 설정했다. 작은 원형 프레임에서 여백이 많아 보이는 문제가 있어, 원본을 백업한 뒤 얼굴이 프레임을 더 채우는 버전으로 크롭했다.

이미지는 캐릭터의 첫인상을 빠르게 전달하는 요소다. 말투만으로는 캐릭터를 이해하기 전까지 시간이 걸리기 때문에, 프로필 사진에서 먼저 `춘배아재`라는 분위기를 전달하도록 했다.

## 답변 방식

답변 방식은 두 갈래다.

1. 포트폴리오 핵심 질문은 고정 답변으로 처리한다.
2. 자유 질문은 LLM을 사용하되, 박춘배 말투 규칙을 항상 유지한다.

고정 답변이라고 해도 문체는 일반 안내문이 아니라 박춘배 캐릭터 말투로 제공한다. 즉, 정보 구조는 안정적으로 유지하되 표현 방식은 캐릭터성을 잃지 않게 한다.

## 이 프로젝트가 보여주는 기획 역량

이 작업은 단순한 챗봇 제작이 아니라, AI 에이전트를 하나의 캐릭터 경험으로 설계한 사례다.

보여주고자 한 역량은 다음과 같다.

- 사용자 언어 관찰: 말투를 감각이 아니라 반복 가능한 지표로 분해했다.
- 캐릭터 UX 설계: 이름, 말투, 프로필, 첫 인사, 답변 톤을 하나의 경험으로 연결했다.
- 정보 구조화: 면접관이 궁금해할 포트폴리오 질문을 먼저 정의했다.
- 하이브리드 응답 설계: 고정 답변과 LLM 답변을 나누어 품질과 효율을 함께 관리했다.
- 안전 기준 수립: 차별적 아재개그와 개인정보 위험을 제거하는 기준을 세웠다.
- 반복 개선: 실제 모바일 사용 중 발생한 커서, 키보드, 스크롤, 고정질문 깨짐 문제를 UX 기준으로 수정했다.

## 면접용 요약

박춘배 챗봇은 제 포트폴리오를 설명하는 AI 에이전트이면서, 동시에 캐릭터 기획 실험입니다. 공개 커뮤니티와 일상 대화에서 관찰한 중장년층 말투의 문체적 특징을 지표화하고, 이를 박춘배라는 가상 캐릭터로 일반화했습니다. 면접관이 궁금해할 질문은 자동완성처럼 제안하고, 핵심 포트폴리오 답변은 고정 답변으로 빠르게 제공하되, 모든 답변은 캐릭터 말투를 유지하도록 설계했습니다. 이후 모바일 사용성, 키보드 동작, 안전하지 않은 아재개그 제거, 프로필 이미지 개선까지 반복하며 AI 에이전트의 말투, 정보 구조, UX 안정성을 함께 다듬었습니다.
